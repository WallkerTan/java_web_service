Phần 1 – Phân tích logic
Nguyên nhân lỗi:  
Phương thức getMovies() đang trả về movies.toString().

Khi gọi toString() trên một List<Movie>, Java sẽ in ra chuỗi mặc định của từng đối tượng, dạng Movie@hashcode.

Đây không phải JSON. Vì vậy client nhận về chuỗi [Movie@3a1b2c3d, Movie@4f2e1a0b] thay vì dữ liệu JSON hợp lệ.

Spring Boot chỉ tự động chuyển đổi sang JSON khi bạn trả về object hoặc collection (ví dụ List<Movie>), chứ không phải String.

Phần 2 – Giải pháp
Để khắc phục, cần:

Trả về danh sách Movie thay vì ép thành chuỗi.  
Spring Boot sẽ dùng Jackson để serialize thành JSON hợp lệ.

Đảm bảo có getters cho các thuộc tính (movieId, title, genre, rating) để Jackson có thể đọc và chuyển đổi.

Giữ nguyên annotation @RestController và @GetMapping("/movies").

Kết quả mong đợi
Khi gọi GET /movies, client sẽ nhận được phản hồi JSON hợp lệ:

json
[
  {
    "movieId": "M001",
    "title": "Inception",
    "genre": "Sci-Fi",
    "rating": 8.8
  },
  {
    "movieId": "M002",
    "title": "Parasite",
    "genre": "Drama",
    "rating": 8.6
  }
]
chốt lại: lỗi nằm ở việc trả về movies.toString(). Chỉ cần trả về List<Movie> (hoặc ResponseEntity<List<Movie>>) thì Spring Boot sẽ tự động convert sang JSON để client parse được.
